# 💕 培培的爱欣食谱 — 给女朋友的点餐小程序

一款温馨可爱的微信点餐小程序，基于微信原生小程序 + 云开发 CloudBase 构建。

## 功能特性

- **用户点餐**：分类浏览菜品、加入购物车、选择口味/香菜/备注、提交订单
- **菜品详情**：查看配料、做法、口味建议
- **管理员后台**：菜品 CRUD、分类管理、订单管理、配置管理、订阅消息
- **订单提醒**：新订单通过微信订阅消息通知管理员

## 技术栈

- 前端：原生微信小程序（WXML + WXSS + JS）
- 后端：微信云开发（云函数 + 云数据库 + 云存储）
- 云函数：login / api / initDemoData

## 如何在微信开发者工具导入

1. 打开微信开发者工具
2. 选择「小程序」→「导入项目」
3. 项目目录选择本项目的根目录
4. AppID 已配置：
5. 点击导入

## 如何填写云环境 ID

1. 在微信开发者工具中，点击顶部工具栏的「云开发」按钮
2. 创建或选择已有的云环境
3. 复制环境 ID
4. 打开 `miniprogram/app.js`，将 `globalData.env` 的值改为你的环境 ID
5. 或在 `project.private.config.json` 中配置

## 如何部署云函数

1. 在微信开发者工具中，展开「cloudfunctions」目录
2. 右键点击以下三个云函数目录，选择「上传并部署：云端安装依赖」：
   - `login`
   - `api`
   - `initDemoData`
3. 等待部署完成（每个约 1-2 分钟）

> **注意**：原有的 `quickstartFunctions` 是 QuickStart 模板的示例，可以保留或删除，不影响本项目功能。

## 如何创建数据库集合

在云开发控制台 → 数据库，创建以下集合：

| 集合名 | 说明 |
|--------|------|
| `app_config` | 全局配置 |
| `categories` | 菜品分类 |
| `dishes` | 菜品 |
| `orders` | 订单 |
| `admins` | 管理员白名单 |
| `admin_subscriptions` | 订阅消息授权 |
| `notification_logs` | 通知发送日志 |
| `operation_logs` | 操作日志（可选） |

> 每个集合的详细字段设计见 [docs/db-schema.md](docs/db-schema.md)。

创建集合后，不需要设置任何索引即可运行（小程序端查询量级较小）。

## 如何运行 initDemoData 初始化数据

1. 确保已部署 `initDemoData` 云函数
2. 确保已创建 `app_config`、`categories`、`dishes` 集合
3. 打开小程序，进入「管理后台」→「设置」页面
4. 点击「初始化演示数据」
5. 等待提示初始化完成

> 也可以直接在云开发控制台 → 云函数中调用 `initDemoData`。

初始化后会创建：
- 4 个分类：主食、小食、饮品、甜品
- 11 道演示菜品
- 全局配置一条

## 如何把我的 openid 加入 admins

### 方法一：通过后台设置页（推荐）

1. 部署所有云函数并初始化数据后
2. 打开小程序，进入「管理后台」→「设置」
3. 页面会显示你的 OpenID
4. 点击「复制」按钮复制 OpenID
5. 打开云开发控制台 → 数据库 → `admins` 集合
6. 点击「添加记录」，填入以下 JSON：

```json
{
  "openid": "你的OpenID",
  "name": "管理员",
  "enabled": true,
  "createdAt": new Date()
}
```

7. 保存后，返回小程序重新进入后台即可

### 方法二：直接调用云函数

在云开发控制台 → 云函数中调用 `api`，传入：
```json
{
  "action": "saveAdminConfig",
  "themeColor": "#FF6B81"
}
```
（此方法仅为演示，实际添加管理员仍需通过数据库操作）

## 如何配置订阅消息模板 ID

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)（小程序管理后台）
2. 进入「功能」→「订阅消息」
3. 在「公共模板库」中搜索合适的订单通知模板，或申请自定义模板
4. 模板内容建议包含：
   - 订单编号（character_string1）
   - 订单状态（thing2）
   - 下单时间（time3）
5. 申请成功后获得模板 ID
6. 进入小程序「管理后台」→「配置」页面
7. 将模板 ID 填入「订阅消息模板 ID」输入框
8. 点击保存
9. 切换到「通知」页，点击「开启订单提醒」授权订阅

> **注意**：开发阶段使用 `TODO_ORDER_TEMPLATE_ID` 占位，需替换为真实模板 ID 才能发送订阅消息。

## 如何测试

### 用户下单测试

1. 打开小程序首页（菜单页）
2. 浏览分类和菜品
3. 点击菜品「加入」按钮，选择口味和香菜偏好
4. 确认加入购物车
5. 点击底部「提交给厨房」
6. 验证成功弹窗和订单号

### 管理员接单测试

1. 确保已将你的 openid 加入 `admins` 集合
2. 进入「管理后台」→「订单」页面
3. 查看新提交的订单
4. 点击订单进入详情
5. 更新订单状态：已接单 → 制作中 → 已完成

### 订阅消息测试

1. 确保已在微信公众平台配置订阅消息模板
2. 在后台「配置」页填入模板 ID 并保存
3. 在后台「通知」页点击「开启订单提醒」
4. 提交一个新订单
5. 验证是否收到微信服务通知

## 常见问题排查

**Q: 页面加载不出来，提示「云函数调用失败」**

A: 检查云函数是否已部署，`app.js` 中的 `env` 是否填写了正确的环境 ID。

**Q: 提交订单后没有收到订阅消息**

A: 检查以下内容：
1. 后台「配置」页是否已填写正确的模板 ID
2. 后台「通知」页是否已点击「开启订单提醒」
3. 微信订阅消息为一次性授权，过期后需重新授权
4. 检查云开发控制台 → `notification_logs` 集合查看错误日志

**Q: 后台管理页面显示「无权限」**

A: 确认你的 openid 已在 `admins` 集合中且 `enabled: true`。可以在后台「设置」页查看和复制 openid。

**Q: 初始化演示数据后没看到菜品**

A: 刷新页面或重新进入。如仍不显示，检查 `categories` 和 `dishes` 集合中是否有数据。

**Q: 上传图片失败**

A: 检查云存储是否已开通，是否有存储空间。
